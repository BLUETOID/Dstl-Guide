Set Theory

What is a Set?

A set is a well-defined collection of distinct objects called elements or members. Sets are usually denoted by capital letters (A, B, C) and elements by lowercase letters (a, b, c).

Notation:
• a ∈ A - 'a' is an element of set A
• a ∉ A - 'a' is not an element of set A
• |A| or n(A) - Cardinality (number of elements)

Types of Sets

  • Empty Set (∅ or {}): Set with no elements
  • Singleton Set: Set with exactly one element
  • Finite Set: Set with countable number of elements
  • Infinite Set: Set with uncountable elements (e.g., ℕ, ℤ, ℝ)
  • Universal Set (U): Set containing all elements under consideration
  • Power Set P(A): Set of all subsets of A, |P(A)| = 2^|A|

Set Operations

• A ∪ B - Union: elements in A OR B (or both)
• A ∩ B - Intersection: elements in BOTH A and B
• A - B or A \ B - Difference: elements in A but not in B
• A' or Aᶜ - Complement: elements NOT in A
• A × B - Cartesian Product: all ordered pairs (a,b)

Important Laws

  • De Morgan's Laws: (A ∪ B)' = A' ∩ B' and (A ∩ B)' = A' ∪ B'
  • Distributive: A ∩ (B ∪ C) = (A ∩ B) ∪ (A ∩ C)
  • Absorption: A ∪ (A ∩ B) = A and A ∩ (A ∪ B) = A

Relations

What is a Relation?

A relation R from set A to set B is a subset of A × B. For binary relations on a single set A, R ⊆ A × A.

Properties of Relations

  • Reflexive: (a, a) ∈ R for all a ∈ A (every element related to itself)
  • Irreflexive: (a, a) ∉ R for any a ∈ A
  • Symmetric: If (a, b) ∈ R then (b, a) ∈ R
  • Antisymmetric: If (a, b) ∈ R and (b, a) ∈ R then a = b
  • Transitive: If (a, b) ∈ R and (b, c) ∈ R then (a, c) ∈ R

Special Relations

• Equivalence Relation: Reflexive + Symmetric + Transitive
• Partial Order: Reflexive + Antisymmetric + Transitive
• Total Order: Partial Order where every pair is comparable

POSET & Lattices

Partially Ordered Set (POSET)

A POSET is a set with a partial order relation (≤). Not all elements need to be comparable. Represented using Hasse diagrams.

Key Terms

  • Maximal element: No element is greater than it
  • Minimal element: No element is less than it
  • Greatest element (top): Greater than or equal to all elements
  • Least element (bottom): Less than or equal to all elements
  • LUB (Supremum): Least Upper Bound of a subset
  • GLB (Infimum): Greatest Lower Bound of a subset

Lattice

A Lattice is a POSET where every pair of elements has:
• A unique LUB (join, denoted a ∨ b)
• A unique GLB (meet, denoted a ∧ b)

Quick Revision Points

  • Power set of n elements has 2ⁿ elements
  • |A × B| = |A| × |B|
  • Equivalence relation partitions a set into equivalence classes
  • Hasse diagram: Remove reflexive loops, transitive edges, direction arrows
  • Boolean lattice: Power set with ⊆ relation

Functions

What is a Function?

A function f: A → B is a relation where every element of A (domain) is related to exactly one element of B (codomain).

Types of Functions

  • Injective (One-to-One): f(a) = f(b) implies a = b. Different inputs → different outputs.
  • Surjective (Onto): Every element in codomain is an image of some element in domain.
  • Bijective: Both injective and surjective. Creates a one-to-one correspondence.
Counting Functions (|A| = m, |B| = n):
• Total functions from A to B: nᵐ
• Injective functions (m ≤ n): n!/(n-m)!
• Bijective functions (m = n): n!

Function Composition

If f: A → B and g: B → C, then (g ∘ f): A → C where (g ∘ f)(x) = g(f(x))

Note: Composition is associative but NOT commutative.

Inverse Function

f⁻¹ exists if and only if f is bijective. Then f⁻¹ ∘ f = identity on A.

Growth of Functions

Big-O Notation

f(n) = O(g(n)) means f grows at most as fast as g
∃c > 0, n₀ such that f(n) ≤ c·g(n) for all n ≥ n₀

Common Growth Rates (Slowest to Fastest)

  • O(1) - Constant
  • O(log n) - Logarithmic
  • O(n) - Linear
  • O(n log n) - Linearithmic
  • O(n²) - Quadratic
  • O(n³) - Cubic
  • O(2ⁿ) - Exponential
  • O(n!) - Factorial

Other Notations

  • Ω(g(n)): Lower bound (at least as fast)
  • Θ(g(n)): Tight bound (exactly as fast)
  • o(g(n)): Strictly slower than
  • ω(g(n)): Strictly faster than

Boolean Algebra

Boolean Operations

  • AND (·): 1 only if both inputs are 1
  • OR (+): 1 if at least one input is 1
  • NOT ('): Flips 0 to 1, 1 to 0
  • XOR (⊕): 1 if inputs are different

Important Laws

• Identity: A + 0 = A, A · 1 = A
• Complement: A + A' = 1, A · A' = 0
• Idempotent: A + A = A, A · A = A
• Absorption: A + AB = A, A(A + B) = A
• De Morgan: (A + B)' = A'B', (AB)' = A' + B'

Canonical Forms

  • SOP (Sum of Products): OR of AND terms (minterms)
  • POS (Product of Sums): AND of OR terms (maxterms)

Karnaugh Maps (K-Maps)

K-Maps are used to simplify Boolean expressions by grouping adjacent 1s (for SOP) or 0s (for POS).

Grouping Rules

  • Groups must be rectangles with size = power of 2 (1, 2, 4, 8...)
  • Groups can wrap around edges
  • Each 1 must be in at least one group
  • Larger groups = simpler terms
  • Overlapping groups are allowed

Quick Revision Points

  • A function must map every domain element to exactly one codomain element
  • Bijection: inverse exists, |A| = |B|
  • O(n²) + O(n) = O(n²) - dominant term wins
  • K-map: 2 variables = 4 cells, 3 variables = 8 cells, 4 variables = 16 cells
  • Gray code ordering in K-maps: only one variable changes between adjacent cells

Propositional Logic

Propositions

A proposition is a declarative statement that is either TRUE or FALSE, but not both.

Logical Connectives

• ¬P (NOT): Negation - flips truth value
• P ∧ Q (AND): True only if both true
• P ∨ Q (OR): True if at least one true
• P → Q (Implication): False only if P true and Q false
• P ↔ Q (Biconditional): True if both same truth value
• P ⊕ Q (XOR): True if exactly one is true

Implication Equivalents

  • P → Q ≡ ¬P ∨ Q
  • Contrapositive: P → Q ≡ ¬Q → ¬P (logically equivalent)
  • Converse: Q → P (NOT equivalent)
  • Inverse: ¬P → ¬Q (NOT equivalent)

Tautology, Contradiction, Contingency

• Tautology: Always TRUE regardless of truth values (e.g., P ∨ ¬P)
• Contradiction: Always FALSE (e.g., P ∧ ¬P)
• Contingency: Sometimes true, sometimes false

Logical Equivalences

  • De Morgan's: ¬(P ∧ Q) ≡ ¬P ∨ ¬Q, ¬(P ∨ Q) ≡ ¬P ∧ ¬Q
  • Double Negation: ¬¬P ≡ P
  • Distributive: P ∧ (Q ∨ R) ≡ (P ∧ Q) ∨ (P ∧ R)
  • Absorption: P ∨ (P ∧ Q) ≡ P

Predicate Logic

Predicates

A predicate P(x) is a statement whose truth depends on the variable x. Example: P(x): "x is even"

Quantifiers

• ∀x P(x) - Universal: "For ALL x, P(x) is true"
• ∃x P(x) - Existential: "There EXISTS some x such that P(x) is true"
• ∃!x P(x) - Unique existential: "There exists exactly one x"

Negation of Quantifiers

  • ¬(∀x P(x)) ≡ ∃x ¬P(x)
  • ¬(∃x P(x)) ≡ ∀x ¬P(x)

Nested Quantifiers

∀x∃y P(x,y) means "For every x, there exists some y (possibly different for each x)"

Order matters! ∀x∃y ≠ ∃y∀x in general

Inference & Proofs

Rules of Inference

• Modus Ponens: P, P→Q ⊢ Q
• Modus Tollens: ¬Q, P→Q ⊢ ¬P
• Hypothetical Syllogism: P→Q, Q→R ⊢ P→R
• Disjunctive Syllogism: P∨Q, ¬P ⊢ Q
• Addition: P ⊢ P∨Q
• Simplification: P∧Q ⊢ P
• Resolution: P∨Q, ¬P∨R ⊢ Q∨R

Proof Techniques

  • Direct Proof: Assume P, derive Q to prove P → Q
  • Contraposition: Prove ¬Q → ¬P instead
  • Contradiction: Assume ¬P, derive a contradiction
  • Mathematical Induction: Base case + inductive step

Quick Revision Points

  • P → Q is only FALSE when P is TRUE and Q is FALSE
  • Contrapositive is always equivalent to original implication
  • De Morgan's: swap AND/OR and negate each part
  • To disprove ∀x P(x), find ONE counterexample
  • To disprove ∃x P(x), show NO x satisfies P(x)

Groups

Definition

A group (G, *) is a set G with operation * satisfying:

1. Closure: a * b ∈ G for all a, b ∈ G
2. Associativity: (a * b) * c = a * (b * c)
3. Identity: ∃e ∈ G such that a * e = e * a = a
4. Inverse: ∀a ∈ G, ∃a⁻¹ such that a * a⁻¹ = a⁻¹ * a = e

Types of Groups

  • Abelian (Commutative) Group: a * b = b * a for all a, b
  • Finite Group: Has finite number of elements
  • Order of Group: |G| = number of elements
  • Order of Element: Smallest positive n where aⁿ = e

Examples of Groups

  • (ℤ, +) - Integers under addition
  • (ℤₙ, +) - Integers mod n under addition
  • (ℝ*, ×) - Non-zero reals under multiplication
  • Symmetric group Sₙ - Permutations of n elements

Subgroups & Cyclic Groups

Subgroup

H is a subgroup of G (H ≤ G) if H ⊆ G and H is itself a group under the same operation.

Subgroup Test: H ≤ G iff:
1. H is non-empty
2. For all a, b ∈ H: a * b⁻¹ ∈ H

Cyclic Groups

G is cyclic if G = ⟨a⟩ = {aⁿ : n ∈ ℤ} for some generator a.

  • Every cyclic group is abelian
  • ℤₙ is cyclic with generator 1
  • Subgroups of cyclic groups are cyclic
  • If |G| = n, then G ≅ ℤₙ or G ≅ ℤ

Cosets

For subgroup H of G and a ∈ G:

  • Left coset: aH = {ah : h ∈ H}
  • Right coset: Ha = {ha : h ∈ H}

Lagrange's Theorem

Lagrange's Theorem:
If H is a subgroup of finite group G, then |H| divides |G|.

Index of H: [G : H] = |G| / |H| = number of distinct cosets

Consequences

  • Order of any element divides order of group
  • For prime p: group of order p has only trivial subgroups {e} and G
  • aᴳ = e for all a ∈ G (where |G| = n)
  • Groups of prime order are cyclic

Rings & Fields

Ring

A ring (R, +, ·) has two operations satisfying:

  • (R, +) is an abelian group
  • Multiplication is associative
  • Distributive laws hold
Types of Rings:
• Commutative Ring: ab = ba
• Ring with Unity: Has multiplicative identity 1
• Integral Domain: Commutative, unity, no zero divisors
• Division Ring: Every non-zero element has multiplicative inverse

Field

A field is a commutative division ring. Examples: ℚ, ℝ, ℂ, ℤₚ (p prime)

Quick Revision Points

  • Group requires 4 properties: closure, associativity, identity, inverse
  • Abelian = commutative
  • Lagrange: |H| divides |G|
  • Every subgroup of a cyclic group is cyclic
  • ℤₚ is a field only when p is prime

Graph Basics

Definition

A graph G = (V, E) consists of a set V of vertices and a set E of edges connecting pairs of vertices.

Handshaking Theorem:
∑ deg(v) = 2|E|
(Sum of all degrees = twice the number of edges)

Corollary: Number of odd-degree vertices is always EVEN

Graph Types

  • Simple Graph: No loops, no multiple edges
  • Multigraph: Multiple edges allowed
  • Directed Graph (Digraph): Edges have direction
  • Weighted Graph: Edges have weights/costs

Special Graphs

  • Complete Kₙ: Every pair connected, edges = n(n-1)/2
  • Cycle Cₙ: n vertices in a cycle, each degree 2
  • Path Pₙ: n vertices in a line, n-1 edges
  • Complete Bipartite Kₘ,ₙ: Two sets, all cross-edges, m×n edges
  • Wheel Wₙ: Cycle with central hub

Paths & Circuits

Definitions

  • Walk: Sequence of vertices with edges between consecutive pairs
  • Trail: Walk with no repeated edges
  • Path: Walk with no repeated vertices
  • Circuit: Closed trail
  • Cycle: Circuit with no repeated vertices (except endpoints)

Euler Paths & Circuits

Euler Path: Visits every EDGE exactly once
Exists iff: Exactly 0 or 2 vertices have odd degree

Euler Circuit: Euler path returning to start
Exists iff: ALL vertices have EVEN degree

Hamiltonian Paths & Cycles

Hamiltonian Path: Visits every VERTEX exactly once
Hamiltonian Cycle: Hamiltonian path returning to start

Dirac's Theorem: If deg(v) ≥ n/2 for all v, Hamiltonian cycle exists
Ore's Theorem: If deg(u) + deg(v) ≥ n for non-adjacent u,v

Planar Graphs

Definition

A graph is planar if it can be drawn in a plane without edge crossings.

Euler's Formula: v - e + f = 2
(vertices - edges + faces = 2, for connected planar graphs)

Corollary: e ≤ 3v - 6 (for v ≥ 3)
For bipartite: e ≤ 2v - 4

Kuratowski's Theorem

G is planar iff G contains no subdivision of K₅ or K₃,₃

  • K₅ is NOT planar (e = 10 > 3(5) - 6 = 9)
  • K₃,₃ is NOT planar (e = 9 > 2(6) - 4 = 8)
  • K₄ IS planar

Trees

Definition

A tree is a connected graph with no cycles.

Tree Properties (for n vertices):
• Exactly n - 1 edges
• Unique path between any two vertices
• Removing any edge disconnects
• Adding any edge creates exactly one cycle

Spanning Tree

Subgraph that includes all vertices and is a tree. MST = Minimum Spanning Tree (minimum total edge weight).

  • Kruskal's: Sort edges, add smallest that doesn't create cycle
  • Prim's: Grow tree from start vertex, add cheapest edge to new vertex

Graph Coloring

Chromatic Number χ(G): Minimum colors needed so no adjacent vertices share a color

• χ(Kₙ) = n
• χ(Cₙ) = 2 (n even), 3 (n odd)
• χ(Tree) = 2
• χ(Bipartite) = 2

Four Color Theorem: Every planar graph can be colored with at most 4 colors.

Combinatorics

Permutations: P(n, r) = n!/(n-r)! (order matters)
Combinations: C(n, r) = n!/(r!(n-r)!) (order doesn't matter)

Pigeonhole Principle:
If n items in k containers and n > k, at least one container has ≥ 2 items
Generalized: At least ⌈n/k⌉ items in some container

Inclusion-Exclusion

|A ∪ B| = |A| + |B| - |A ∩ B|
|A ∪ B ∪ C| = |A| + |B| + |C| - |A∩B| - |B∩C| - |A∩C| + |A∩B∩C|

Quick Revision Points

  • Euler = edges, Hamiltonian = vertices
  • Tree: n vertices → n-1 edges
  • Euler's formula: v - e + f = 2
  • K₅ and K₃,₃ are non-planar
  • Planar graph χ ≤ 4 (Four Color Theorem)
  • Pigeonhole: more items than containers → sharing occurs
