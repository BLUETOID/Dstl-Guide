Functions

Definition: Function

A function f from set A to set B (f: A → B) is a relation where each element in A is associated with exactly one element in B.

Domain: Set A (input set)
Codomain: Set B (possible outputs)
Range: {f(a) | a ∈ A} ⊆ B (actual outputs)

Function vs Not a Function

   FUNCTION                    NOT A FUNCTION              NOT A FUNCTION
   (each input → one output)   (one input → two outputs)   (input has no output)
   
   A       B                   A       B                   A       B
   ┌───┐   ┌───┐               ┌───┐   ┌───┐               ┌───┐   ┌───┐
   │ 1 │──→│ a │               │ 1 │──→│ a │               │ 1 │──→│ a │
   │ 2 │──→│ b │               │ 2 │╲  │ b │               │ 2 │   │ b │
   │ 3 │──→│ b │               │   │ ╲→│ c │               │ 3 │──→│ c │
   └───┘   └───┘               └───┘   └───┘               └───┘   └───┘
      ✓ Valid                     ✗ Invalid                   ✗ Invalid

Function Notation

f: A → B
f(x) = expression (e.g., f(x) = x²)
f: x ↦ f(x) (maps x to f(x))

Example

Let f: ℤ → ℤ where f(x) = x². Find domain, codomain, range.

Domain: ℤ (all integers)
Codomain: ℤ (all integers)
Range: {0, 1, 4, 9, 16, ...} = {n² | n ∈ ℤ}
Note: Range ⊂ Codomain (not all integers are squares)

Types of Functions

Injective (One-to-One)

f(a) = f(b) ⟹ a = b
Different inputs → different outputs
No two elements map to same output

Surjective (Onto)

∀b ∈ B, ∃a ∈ A: f(a) = b
Every element in B is mapped to
Range = Codomain

Bijective (One-to-One & Onto)

Both injective AND surjective
Perfect pairing between A and B
Has an inverse function

INJECTIVE (1-1)           SURJECTIVE (Onto)         BIJECTIVE (1-1 & Onto)
A       B                 A       B                 A       B
┌───┐   ┌───┐             ┌───┐   ┌───┐             ┌───┐   ┌───┐
│ 1 │──→│ a │             │ 1 │──→│ a │             │ 1 │──→│ a │
│ 2 │──→│ b │             │ 2 │╲  │   │             │ 2 │──→│ b │
│ 3 │──→│ c │             │ 3 │─╲→│ b │             │ 3 │──→│ c │
└───┘   │ d │←(unused)    └───┘   └───┘             └───┘   └───┘
        └───┘             Every b covered           Perfect matching

Testing Injectivity & Surjectivity

PropertyTestFor f: A → B
InjectiveHorizontal line test (graphs) OR f(a)=f(b)⟹a=b|A| ≤ |B| needed
SurjectiveEvery b has preimage OR Range = B|A| ≥ |B| needed
BijectiveBoth above|A| = |B| needed

Example: Classify Functions

f: ℝ → ℝ, f(x) = 2x + 3

Injective? f(a) = f(b) ⟹ 2a+3 = 2b+3 ⟹ a = b ✓
Surjective? For any y ∈ ℝ, x = (y-3)/2 gives f(x) = y ✓
Conclusion: f is bijective ✓

Special Functions

Floor Function ⌊x⌋

Greatest integer ≤ x
⌊3.7⌋ = 3, ⌊-2.3⌋ = -3

Ceiling Function ⌈x⌉

Smallest integer ≥ x
⌈3.2⌉ = 4, ⌈-2.7⌉ = -2

Identity Function

I(x) = x for all x
Maps every element to itself

Constant Function

f(x) = c for all x
Every input maps to same output

Composition & Inverse

Definition: Composition

If f: A → B and g: B → C, then g ∘ f: A → C
(g ∘ f)(x) = g(f(x))
"Apply f first, then g"

Composition g ∘ f:   A ──f──→ B ──g──→ C
                     x ──────→ f(x) ──→ g(f(x))

Important

Composition is NOT commutative: g ∘ f ≠ f ∘ g (generally)
Composition IS associative: (h ∘ g) ∘ f = h ∘ (g ∘ f)

Example

f(x) = x + 2, g(x) = x². Find g ∘ f and f ∘ g.

(g ∘ f)(x) = g(f(x)) = g(x+2) = (x+2)² = x² + 4x + 4
(f ∘ g)(x) = f(g(x)) = f(x²) = x² + 2

Note: g ∘ f ≠ f ∘ g

Inverse Function

Definition

For bijective f: A → B, the inverse f⁻¹: B → A satisfies:
f⁻¹(f(x)) = x and f(f⁻¹(y)) = y
f⁻¹ ∘ f = Iₐ and f ∘ f⁻¹ = Iᵦ

Key Facts

• Only bijective functions have inverses
• (f⁻¹)⁻¹ = f
• (g ∘ f)⁻¹ = f⁻¹ ∘ g⁻¹

Example: Find Inverse

f(x) = 3x + 5. Find f⁻¹(x).

Let y = 3x + 5
Solve for x: y - 5 = 3x ⟹ x = (y-5)/3
f⁻¹(x) = (x - 5)/3

Verify: f(f⁻¹(x)) = f((x-5)/3) = 3·(x-5)/3 + 5 = x ✓

Growth of Functions (Asymptotic Notation)

Big-O Notation O(g(n))

f(n) = O(g(n)) if ∃ positive constants c and n₀ such that:
0 ≤ f(n) ≤ c·g(n) for all n ≥ n₀
"f grows at most as fast as g" (upper bound)

Big-Ω Notation Ω(g(n))

f(n) = Ω(g(n)) if ∃ positive constants c and n₀ such that:
0 ≤ c·g(n) ≤ f(n) for all n ≥ n₀
"f grows at least as fast as g" (lower bound)

Big-Θ Notation Θ(g(n))

f(n) = Θ(g(n)) if f(n) = O(g(n)) AND f(n) = Ω(g(n))
"f grows exactly as fast as g" (tight bound)

Common Growth Rates (Slowest to Fastest)

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
NotationNameExample
O(1)ConstantArray access
O(log n)LogarithmicBinary search
O(n)LinearLinear search
O(n log n)LinearithmicMerge sort
O(n²)QuadraticBubble sort
O(2ⁿ)ExponentialSubset generation

Example: Prove Big-O

Show that 3n² + 5n + 2 = O(n²)

Need to find c, n₀ such that 3n² + 5n + 2 ≤ cn² for n ≥ n₀

For n ≥ 1: 5n ≤ 5n² and 2 ≤ 2n²
So: 3n² + 5n + 2 ≤ 3n² + 5n² + 2n² = 10n²

Choose c = 10, n₀ = 1
Therefore, 3n² + 5n + 2 = O(n²) ✓

Boolean Algebra

Definition

Boolean Algebra is an algebraic structure (B, +, ·, ', 0, 1) where:
B = {0, 1}, + = OR, · = AND, ' = NOT
Used in digital logic, computer circuits, and set theory.

Boolean Operations Truth Table

ABA + B (OR)A · B (AND)A' (NOT)A ⊕ B (XOR)
000010
011011
101001
111100

Laws of Boolean Algebra

Identity: A + 0 = A, A · 1 = A
Null: A + 1 = 1, A · 0 = 0
Idempotent: A + A = A, A · A = A
Complement: A + A' = 1, A · A' = 0
Involution: (A')' = A
Commutative: A + B = B + A, A · B = B · A
Associative: (A + B) + C = A + (B + C)
Distributive: A · (B + C) = A·B + A·C
Absorption: A + A·B = A, A · (A + B) = A
De Morgan: (A + B)' = A' · B', (A · B)' = A' + B'

Boolean Expressions

Canonical Forms

SOP (Sum of Products): OR of AND terms (minterms)
Example: F = A'B + AB' + AB = Σm(1,2,3)

POS (Product of Sums): AND of OR terms (maxterms)
Example: F = (A+B)(A+B')(A'+B) = ΠM(0)

Example: Simplify

Simplify: F = AB + AB' + A'B

F = AB + AB' + A'B
= A(B + B') + A'B   (factoring A)
= A(1) + A'B   (complement law)
= A + A'B
= A + B   (absorption: A + A'B = A + B)
F = A + B

Karnaugh Maps (K-Maps)

Definition

A K-Map is a visual method for simplifying Boolean expressions. Adjacent cells differ by exactly one variable (Gray code ordering).

2-Variable K-Map

          B=0   B=1
        ┌─────┬─────┐
   A=0  │ m₀  │ m₁  │     m₀ = A'B'
        ├─────┼─────┤     m₁ = A'B
   A=1  │ m₂  │ m₃  │     m₂ = AB'
        └─────┴─────┘     m₃ = AB

3-Variable K-Map

              BC
           00  01  11  10
         ┌────┬────┬────┬────┐
    A=0  │ m₀ │ m₁ │ m₃ │ m₂ │
         ├────┼────┼────┼────┤
    A=1  │ m₄ │ m₅ │ m₇ │ m₆ │
         └────┴────┴────┴────┘
         Gray code: 00→01→11→10

K-Map Simplification Rules

  1. Group adjacent 1s in powers of 2 (1, 2, 4, 8...)
  2. Groups can wrap around edges
  3. Make groups as large as possible
  4. Every 1 must be in at least one group
  5. Overlapping groups are allowed
  6. Variables that stay constant in a group remain in the term

Example: 3-Variable K-Map

Simplify F(A,B,C) = Σm(0,1,2,4,5,6)

              BC
           00  01  11  10
         ┌────┬────┬────┬────┐
    A=0  │ 1  │ 1  │ 0  │ 1  │
         ├────┼────┼────┼────┤
    A=1  │ 1  │ 1  │ 0  │ 1  │
         └────┴────┴────┴────┘
Groups:
• Column BC=00 (m₀,m₄): gives B'C'
• Column BC=01 (m₁,m₅): gives B'C
• Column BC=10 (m₂,m₆): gives BC'

F = B'C' + B'C + BC' = B' + C'
